# Каприз — интернет-магазин женской одежды

Полноценный каркас интернет-магазина женской одежды: каталог с фильтрами, страницы товаров с галереей и отзывами, личный кабинет с избранным и админ-панель. Сделан в стиле реальных fashion-магазинов (Zara / ASOS / Shopify-витрины).

## Стек

- **Node.js + Express** — сервер
- **EJS + express-ejs-layouts** — серверный рендеринг страниц
- **SQLite (better-sqlite3)** — база данных
- **express-session + bcrypt** — авторизация и хеширование паролей
- Чистый CSS + ванильный JS (без фреймворков), шрифты Playfair Display + Manrope

## Запуск

```bash
npm install      # установка зависимостей
npm run seed     # наполнение базы (категории, товары, отзывы, админ)
npm run dev      # запуск с автоперезагрузкой (nodemon)
# или
npm start        # обычный запуск
```

Открыть: http://localhost:3000

## Доступ администратора

Логин и пароль задаются в `.env` (`ADMIN_EMAIL`, `ADMIN_PASSWORD`):

- email: `admin@kapriz.shop`
- пароль: `admin123`

После входа администратору доступна панель по адресу `/admin`.

## Деплой на Render (бесплатно)

Проект готов к деплою на [Render](https://render.com) — даёт бесплатный адрес вида `https://kapriz.onrender.com`. В репозитории уже есть `render.yaml` (Blueprint) и авто-наполнение базы при первом старте.

**Шаги:**

1. **Залить код на GitHub.** Создай пустой репозиторий на github.com, затем в папке проекта:
   ```bash
   git init
   git add .
   git commit -m "Каприз — интернет-магазин"
   git branch -M main
   git remote add origin https://github.com/<твой-логин>/<репозиторий>.git
   git push -u origin main
   ```
2. **Создать сервис на Render.** Зайти на render.com → **New** → **Blueprint** → подключить свой GitHub-репозиторий. Render сам прочитает `render.yaml`.
3. **Задать пароль админа.** При создании Render спросит значение `ADMIN_PASSWORD` (поле помечено `sync: false`) — введи свой пароль. `SESSION_SECRET` сгенерируется автоматически.
4. **Дождаться сборки** (~2–4 минуты). Готово — сайт по адресу `https://<имя>.onrender.com`.

**Нюансы бесплатного тарифа Render:**
- Сервис «засыпает» после 15 минут без трафика; первый запрос после сна грузится ~30 секунд.
- Диск эфемерный: при перезапуске база пересоздаётся. **Каталог восстанавливается автоматически** (авто-сид), админ берётся из переменных окружения. Но **новые регистрации пользователей и добавленные через админку товары не сохранятся** после перезапуска.
- Чтобы данные сохранялись навсегда — нужен платный план с persistent disk: в `render.yaml` раскомментируй блок `disk` и добавь переменную `DATA_DIR=/data` (тогда SQLite-файл ляжет на постоянный диск).

**Свой домен** (опционально): купить у любого регистратора и привязать в Render → Settings → Custom Domains (Render бесплатно выдаст SSL).

## Возможности

**Витрина**
- Главная: hero (с реальными метриками), bento-сетка категорий, новинки, хиты, распродажа, блок о бренде, рассылка
- Поиск по каталогу (поле в шапке)
- Каталог: фильтры по категории, цене, размеру, цвету, распродаже + сортировка, активные чипы фильтров со сбросом, пагинация «Показать ещё»
- Наличие товара: бейджи «Нет в наличии» / «Мало», индикатор остатка на карточке товара
- Карточка товара: галерея, выбор размера (обязателен) и цвета, таблица размеров (модалка), рейтинг, отзывы с формой, похожие, «вы недавно смотрели»
- Корзина: добавление в корзину, выезжающая панель (slide-cart), страница корзины с расчётом доставки (оформление — демонстрационное, без онлайн-оплаты)
- Избранное (wishlist) с переключением по «сердечку» без перезагрузки (AJAX)
- Регистрация / вход / личный кабинет (профиль, избранное, настройки: редактирование профиля и смена пароля)
- Страницы: «О бренде», «Контакты», «Доставка и оплата», «Обмен и возврат», «Частые вопросы»

**Безопасность и доступность**
- Защита от open-redirect при входе, усиленные cookie сессии (httpOnly/sameSite), ограничение попыток входа
- Семантическая разметка, `:focus-visible`, `aria`-атрибуты, `prefers-reduced-motion`, локальные изображения с фолбэком

**Админ-панель (`/admin`)**
- Дашборд со статистикой (товары, пользователи, отзывы, средний рейтинг, заканчивающиеся остатки)
- Управление товарами: создание / редактирование / удаление, поиск и фильтр по категории, поле остатка, галерея из нескольких фото
- Управление категориями: создание, редактирование, удаление
- Модерация отзывов
- Список пользователей

## Структура

```
app.js, server.js        — конфигурация и запуск Express
db/                       — схема и подключение SQLite
seed/                     — наполнение базы (catalogData.js + seed.js)
models/                   — слой работы с БД
middleware/               — авторизация и текущий пользователь
routes/                   — маршруты
views/                    — EJS-шаблоны (layouts, partials, страницы, admin)
public/css, public/js     — стили и клиентский JS
public/images/catalog/    — фотографии товаров
utils/format.js           — общие хелперы (цены, цвета, изображения)
```

## Заметки

- Фотографии товаров лежат локально в `public/images/catalog/` — сайт не зависит от внешних сервисов и грузится мгновенно.
- Оформление покупки (корзина/оплата) сознательно вне текущего этапа — это витрина-каркас.
